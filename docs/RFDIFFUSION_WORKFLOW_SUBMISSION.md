# RFdiffusion workflow submission contract

This is the fixed path for running RFdiffusion from the Workbench UI.

## 1. Generate or open the workflow

- A project workflow should contain an RFdiffusion node before downstream ProteinMPNN, folding, scoring, and candidate review nodes.
- The RFdiffusion node must use the registered `plugin_rfdiffusion` model plugin.
- The node inputs are resolved from explicit node input artifacts plus upstream workflow edges. The staged input contract is always `input/manifest.json`.
- The RFdiffusion output contract is `output/manifest.json`; generated PDBs are registered as backbone artifacts and then become generated candidates.

## 2. Edit RFdiffusion node parameters

The Node details panel is the source of truth before submission. Save or preview with these parameters:

- `scaffold`: route label such as `monellin` or `brazzein`; this is written to candidate metadata.
- `contigmap.contigs`: RFdiffusion Hydra contig expression.
- `ppi.hotspot_res`: optional hotspot residue list.
- `inference.num_designs`: number of RFdiffusion trajectories/backbones.
- `inference.output_prefix`: output PDB prefix. The cluster script normalizes this to a basename for the staged output folder.
- `diffuser.partial_T`: set to `0` for unconditional/design-from-contig runs; set above `0` for partial diffusion from a compatible starting structure.
- `contigmap.provide_seq`: preserved sequence ranges for partial diffusion. Single residue lists such as `[1,13,19]` are normalized to `[1-1,13-13,19-19]` before command rendering.
- Advanced controls include `diffuser.T`, denoiser noise scales, inpaint ranges, checkpoint override, symmetry, and guiding potentials.

## 3. Preview the exact LSF script

Use **Generate script** in Node details before submitting. The preview is generated by the same backend renderer used by cluster submission, so the displayed script and submitted script do not drift.

The preview shows:

- LSF queue/resource/GPU directives.
- The staged remote job directory under the configured LSF root.
- The RFdiffusion conda environment and `run_inference.py` command.
- The exact Hydra arguments generated from the current node parameters.
- The post-run manifest writer that records generated backbones and `rfdiffusion_run.json`.

The script can be copied or downloaded for manual review.

## 4. Submit to the cluster

Use **Manual LSF submit** in the Jobs panel after selecting the RFdiffusion node.

- Queue/resource/GPU overrides are optional. If no queue is supplied, the RFdiffusion renderer chooses `4v100-16-e5` for standard runs and `8v100-32-sc` when `diffuser.partial_T > 0`.
- Submission creates a new job, stages `input/`, writes `work/submit.lsf`, uploads the workspace, and runs `bsub < work/submit.lsf`.
- The backend stores the scheduler id, polls status, and keeps the log tail visible in the Jobs panel.
- If parameters are changed in Node details, the submit call sends those current parameters and persists them on the node before staging.

## 5. Collect outputs and continue

After the LSF job finishes, click **Sync result** in Jobs.

- The backend downloads the remote `output/` directory.
- `output/manifest.json` is parsed.
- PDB outputs become artifacts and are registered in the Candidate table with `generated_backbone` status.
- Generated-but-unscored RFdiffusion candidates are valid candidate rows; downstream ProteinMPNN, folding, and scoring steps add sequences and scores later.

## Sweet protein reference routes

- Monellin route: uses the prepared Monellin structure, RFdiffusion contigs that preserve the known sweet-protein fold context, and standard generation with `diffuser.partial_T = 0`.
- Brazzein route: uses partial diffusion from the prepared Brazzein structure, length-compatible contigs, and `contigmap.provide_seq` to keep conserved residues visible.
- The route label must be set in `scaffold` so generated candidates can be grouped in the table.

## Review checklist before submitting

- The selected node is the intended RFdiffusion node.
- The staged input artifact is the intended PDB.
- `contigmap.contigs` is compatible with the input chain/residue numbering.
- `diffuser.partial_T` matches the run type.
- `contigmap.provide_seq` is only used for partial diffusion and uses valid zero-indexed RFdiffusion ranges.
- `inference.num_designs` is set to the intended batch size.
- The generated script preview contains the expected queue, environment, RFdiffusion path, output prefix, and Hydra arguments.
