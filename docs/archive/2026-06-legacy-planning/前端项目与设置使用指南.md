# BDA 前端项目与设置使用指南

## 1. 页面关系

`Experiments` 是实验项目入口。一个实验项目包含：

- 工作流和计算任务。
- 上传的靶点、结构、序列与约束文件。
- 生成的候选物。
- 湿实验结果。
- 报告、结构包、表格和最终交付包。

`Workflow`、`Candidates` 和 `Results` 不再是彼此独立的数据空间，它们始终使用当前实验项目。

## 2. 选择或创建项目

进入 `Experiments` 后：

1. 在“实验项目”区域选择已有项目。
2. 或点击“新建项目”。
3. 填写项目名称、项目类型与目标说明。
4. 点击“创建并选择”。

创建成功后，该项目成为当前项目。

如果未选择项目便进入 `Workflow`、`Candidates` 或 `Results`，系统会先显示项目选择器，不会默认混入 PD-1 演示数据。

## 3. 项目上下文

项目选择会同时保存到：

- 页面 URL 的 `project` 参数。
- 前端持久状态。

因此从工作流进入候选物或结果页面时，当前项目保持不变。各页面顶部会显示“当前实验项目”。

## 4. 应用设置

点击顶部齿轮按钮进入“应用设置与连接验证”。

### 应用模式

- 可创建新项目。
- 可编辑工作流。
- 可提交真实集群任务。
- 候选物、结果和交付数据写入当前项目。

### 演示模式

- 自动选择 PD-1 预置演示项目。
- 以只读方式浏览预置工作流和结果。
- 禁止创建项目。

## 5. 连接验证

设置中心可以检查：

- BDA FastAPI 后端。
- 南科大 LSF 集群及当前 CPU/GPU 队列。
- DeepSeek/OpenAI-compatible API。

集群验证当前检查：

- `gpu-bme-liz`
- `v3-64`

DeepSeek 需要填写 API Key 后才能点击 `Test API`。

## 6. 推荐操作顺序

1. 打开设置并确认处于“应用模式”。
2. 验证后端、集群和 DeepSeek。
3. 在 `Experiments` 选择或创建项目。
4. 进入 `Workflow` 上传输入并建立路线。
5. 提交计算并查看日志、产物。
6. 在 `Candidates` 审核候选物。
7. 在 `Results` 上传实验数据并生成交付包。
